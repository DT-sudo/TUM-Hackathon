# Model Contract — Traffic Forecasting Pipeline

Interface between:
- `scripts/train_model.py` → produces the bundle
- `models/prediction_pipeline.joblib` → the serialized bundle
- `backend/app/processors/traffic_analyzer.py` → loads and calls the bundle
- `frontend/` → displays results via API

---

## Model File

```
models/prediction_pipeline.joblib
```

Generated by `python scripts/train_model.py` **before** starting the application.
The application loads this file at startup — it does **not** train.

---

## Approach (v2.0.0)

A single **regressor** predicts slot volume (`kfz_h_slot`). All traffic
categories (colours) are then **derived from the predicted volume** using
percentile thresholds:

- **Slot category** → slot volume vs per-`(corridor, direction, slot)` thresholds
- **Daily category** → daily total (sum of 6 slot volumes) vs per-`(corridor, direction)` thresholds

This guarantees the colour shown always matches the vehicle count shown, and
gives the daily calendar a realistic spread (≈40/20/20/15/5 %). The earlier
v1 design used a separate classifier plus a max-of-slots daily rule, which
marked ~45 % of days red — that classifier has been removed.

---

## Bundle Structure

The `.joblib` file contains a Python dict with these keys:

```python
bundle = joblib.load("models/prediction_pipeline.joblib")

bundle["regressor"]           # sklearn Pipeline → predicts kfz_h_slot (float, vehicles)
bundle["slot_thresholds"]     # dict: key = "A8E|outbound|3" → [p40, p60, p80, p95]
bundle["daily_thresholds"]    # dict: key = "A8E|outbound"   → [p40, p60, p80, p95]
bundle["category_thresholds"] # backward-compat alias of slot_thresholds
bundle["baselines"]           # dict of historical averages for future-date features
bundle["feature_columns"]     # list of 25 feature names in the required order
bundle["training_info"]       # metadata + honest year-ahead evaluation metrics
```

---

## What the Pipeline Includes

The regressor pipeline contains:
1. `StandardScaler` — normalises all 25 numerical features
2. `GradientBoostingRegressor` — 400 estimators, depth 5, lr 0.05, subsample 0.8

No categorical encoding needed: corridor and direction are already binary flags.

---

## Deriving a Category from Volume

```python
from app.processors.feature_builder import assign_category

vol = bundle["regressor"].predict(input_df)[0]          # e.g. 13118 vehicles
slot_thr  = bundle["slot_thresholds"]["A8E|outbound|3"]  # [p40,p60,p80,p95]
category  = assign_category(vol, slot_thr)               # int 1–5

# Daily colour: sum the 6 predicted slot volumes, then:
daily_thr = bundle["daily_thresholds"]["A8E|outbound"]
daily_cat = assign_category(daily_total, daily_thr)
```

## Honest Evaluation (year-ahead holdout)

`training_info["evaluation"]` stores metrics from training on 2023–2024 and
testing on the held-out 2025 (this is how the model performs forecasting an
unseen year — the realistic measure of quality):

| Metric | Value |
|--------|-------|
| Slot volume R² | ~0.93 |
| Slot volume MAE | ~520 vehicles |
| Daily category accuracy | ~0.65 |
| Daily category within ±1 shade | ~0.96 |

The final shipped model is then retrained on all of 2023–2025.

## Feature Input (25 columns, all numerical)

One row = one **(date × time_slot × corridor × direction)** combination.

| # | Feature | Type | Range | Description |
|---|---------|------|-------|-------------|
| 1 | `month` | int | 1–12 | Calendar month |
| 2 | `day_of_week` | int | 0–6 | Mon=0, Sun=6 |
| 3 | `week_of_year` | int | 1–53 | ISO week number |
| 4 | `time_slot` | int | 1–6 | 1=night, 2=morning, 3=midday, 4=afternoon, 5=evening, 6=late |
| 5 | `is_weekend` | int | 0/1 | Saturday or Sunday |
| 6 | `is_public_holiday_de` | int | 0/1 | German national public holiday |
| 7 | `is_public_holiday_bavaria` | int | 0/1 | Bavarian-specific public holiday |
| 8 | `is_bridge_day` | int | 0/1 | Brückentag between holiday and weekend |
| 9 | `is_long_weekend` | int | 0/1 | 3+ consecutive non-workdays |
| 10 | `is_school_holiday_bavaria` | int | 0/1 | Bavaria school holiday active |
| 11 | `is_school_holiday_bw` | int | 0/1 | Baden-Württemberg school holiday active |
| 12 | `school_holiday_overlap` | int | 0–2 | Sum of BY + BW indicators |
| 13 | `days_until_school_holiday` | int | 0–30 | Days until next BY holiday starts (0 if in holiday; clipped at 30) |
| 14 | `days_since_school_holiday` | int | 0–30 | Days since last BY holiday ended (0 if in holiday; clipped at 30) |
| 15 | `is_summer_season` | int | 0/1 | June–September |
| 16 | `is_winter_sports_season` | int | 0/1 | December–March |
| 17 | `is_easter_period` | int | 0/1 | ±4 days around Easter Sunday |
| 18 | `is_christmas_period` | int | 0/1 | Dec 22 – Jan 6 |
| 19 | `is_outbound` | int | 0/1 | 1 = outbound (→ Austria), 0 = inbound |
| 20 | `is_a93` | int | 0/1 | 1 = A93 South, 0 = A8 East |
| 21 | `hist_kfz_dow_slot` | float | ≥0 | Historical mean kfz_h_slot for same (corridor, direction, day_of_week, slot) |
| 22 | `hist_kfz_month_slot` | float | ≥0 | Historical mean for same (corridor, direction, month, slot) |
| 23 | `hist_sv_share` | float | 0–1 | Historical heavy vehicle share for this corridor/direction |
| 24 | `clim_air_temp_c` | float | −10–35 | Climatological (or actual) air temperature for that day/month |
| 25 | `is_frost_risk_month` | int | 0/1 | Month historically has road surface temp < 2°C |

## Loading and Running Predictions

```python
import joblib
import pandas as pd
from pathlib import Path

bundle = joblib.load(Path("models/prediction_pipeline.joblib"))
feature_columns = bundle["feature_columns"]  # ordered list of 25 column names

input_df = pd.DataFrame([{
    "month": 8, "day_of_week": 5, "week_of_year": 31,
    "time_slot": 3, "is_weekend": 1,
    "is_public_holiday_de": 0, "is_public_holiday_bavaria": 0,
    "is_bridge_day": 0, "is_long_weekend": 0,
    "is_school_holiday_bavaria": 1, "is_school_holiday_bw": 1,
    "school_holiday_overlap": 2,
    "days_until_school_holiday": 0, "days_since_school_holiday": 0,
    "is_summer_season": 1, "is_winter_sports_season": 0,
    "is_easter_period": 0, "is_christmas_period": 0,
    "is_outbound": 1, "is_a93": 0,
    "hist_kfz_dow_slot": 38500.0, "hist_kfz_month_slot": 35200.0,
    "hist_sv_share": 0.065, "clim_air_temp_c": 21.0,
    "is_frost_risk_month": 0,
}])[feature_columns]  # enforce column order

volume   = bundle["regressor"].predict(input_df)[0]           # estimated vehicles in slot
category = assign_category(                                    # int 1–5, derived from volume
    volume, bundle["slot_thresholds"]["A8E|outbound|3"]
)
```

**Important:** always use `input_df[feature_columns]` to enforce column order before calling `predict()`.

## `baselines` dict — structure for future-date inference

```python
bundle["baselines"] = {
    # (corridor, direction, day_of_week, time_slot) → mean kfz_h_slot
    "by_dow_slot": {
        ("A8E", "outbound", 5, 3): 38500.0,   # Saturday midday outbound A8E
        ...
    },
    # (corridor, direction, month, time_slot) → mean kfz_h_slot
    "by_month_slot": {
        ("A8E", "outbound", 8, 3): 35200.0,   # August midday outbound A8E
        ...
    },
    # (corridor, direction) → mean sv_share
    "sv_share": {
        ("A8E", "outbound"): 0.065,
        ...
    },
    # month → climatological air temp °C
    "clim_air_temp": {1: 1.0, 2: 2.0, ..., 8: 20.0},
    # set of months historically having frost risk
    "frost_risk_months": {1, 2, 11, 12},
}
```

These baselines are computed from 2023–2025 training data and allow the backend to build feature vectors for any 2026+ date without needing the raw traffic files at runtime.

## Error Cases

| Error | Cause | Fix |
|-------|-------|-----|
| `FileNotFoundError` | `.joblib` not found | `python scripts/train_model.py` |
| `ValueError: corridor must be` | Invalid corridor | Use `"A8E"` or `"A93S"` |
| `ValueError: direction must be` | Invalid direction | Use `"outbound"` or `"inbound"` |
| `KeyError` / column mismatch | Feature column missing or wrong order | Use `df[feature_columns]` with bundle's list |
| `503` from `/api/forecast` | Model not loaded | Run `train_model.py`, restart backend |

## Model Version

`2.0.0` — GradientBoosting regressor + percentile-threshold category derivation
(slot & daily), trained on A8E/A93S hourly detector data 2023–2025, 25 features.
Supersedes `1.0.0` (separate classifier + max-of-slots daily rule, removed).

Retrain when: new yearly data is available, or additional features (roadworks,
weather forecast) are added.
