# TUM Science Hackathon 2026

AI-powered dataset analysis platform. Theme-independent architecture — adapts to any of the six challenge topics in ~15 minutes.

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Backend | Python 3.11 · FastAPI · uvicorn |
| Data | pandas · numpy |
| ML | scikit-learn · joblib |
| Frontend | React 19 · TypeScript 5.8 (Lovable) |
| UI | shadcn/ui · Radix UI · Tailwind CSS 4 · Recharts |
| Routing | TanStack Router · TanStack React Query |
| Build | Vite 8 · Bun |

## Quick Start

### 1. Clone and set up backend

```bash
git clone https://github.com/DT-sudo/TUM-Hackathon.git
cd TUM-Hackathon

python -m venv .venv
source .venv/bin/activate    # Windows: .venv\Scripts\activate
pip install -r backend/requirements.txt
```

### 2. Configure environment

```bash
cp .env.example .env
# Edit .env if needed (default PORT=8000)
```

### 3. Download dataset (hackathon day)

```bash
bash scripts/download_data.sh <GOOGLE_DRIVE_FILE_ID>
```

### 4. Prepare data and train model

```bash
python scripts/clean_data.py
python scripts/build_features.py
python scripts/train_model.py
# Generates: models/prediction_pipeline.joblib
```

### 5. Run backend

```bash
uvicorn backend.app.main:app --reload --port 8000
# API:  http://localhost:8000
# Docs: http://localhost:8000/docs
```

### 6. Run frontend

The React frontend is generated by Lovable and lives in `frontend/`.

```bash
cd frontend
bun install
bun dev
# UI: http://localhost:5173
```

## Architecture

```
data/raw/  →  scripts/  →  models/*.joblib  →  backend/app/  →  frontend/
(datasets)    (prepare)    (trained pipeline)  (FastAPI API)     (Lovable UI)
```

Frontend fetches results from the backend via JSON. CORS is open for local development.

## Adapting to the Hackathon Theme

Once the theme is announced, update these four files (~15 min):

1. `docs/THEME_SPECIFIC.md` — dataset structure, success criteria
2. `docs/MODEL_CONTRACT.md` — model input/output interface
3. `backend/app/processors/analyzer.py` — theme-specific processing logic
4. `backend/app/api/routes.py` — theme-specific endpoints
5. `CLAUDE.md` — Mission section

Everything else stays unchanged.
