## What Changes on Hackathon Day (takes ~15 minutes)

1. **Learn the theme at 9:00 AM**

2. **Fill `docs/THEME_SPECIFIC.md`** — dataset structure, columns, success criteria

3. **Fill `docs/MODEL_CONTRACT.md`** — model input/output interface

4. **Update `CLAUDE.md` Mission section** — one sentence describing the specific task

5. **Commit and push:**
   ```bash
   git add docs/ CLAUDE.md
   git commit -m "docs: theme details — [theme name]"
   git push origin dev
   ```

6. **Team starts work (10:00 AM):**

   Backend developer:
   ```bash
   cd backend && python -m venv .venv && source .venv/bin/activate
   pip install -r requirements.txt
   uvicorn backend.app.main:app --reload --port 8000
   # Implement analyzer.py and routes.py for the theme
   ```

   Frontend developer:
   ```
   1. Generate UI in Lovable for the specific theme
   2. Connect fetch() calls to http://localhost:8000/api/...
   3. Export to frontend/ folder
   ```

   Data/ML role:
   ```bash
   bash scripts/download_data.sh <GOOGLE_DRIVE_FILE_ID>
   python scripts/clean_data.py
   python scripts/build_features.py
   python scripts/train_model.py
   ```

```bash

git checkout dev

# After changes
git add .

# End of hackathon — merge to main
git checkout main
git merge dev
git push origin main
```

## Universal Principle

```
ANY theme requires the same three steps:
──────────────────────────────────────
1. GET the dataset      → data/raw/
2. PROCESS it           → backend/app/processors/
3. DISPLAY the results  → frontend/ (via API)
──────────────────────────────────────
This works for all 6 themes.
```
